6.5 Transform vs Lerp

Lastly, I want to talk about when to use Transform vs Lerp. Transfomr's syntax is nicer (since we dont need to specify start), but it gives less control. We use transform whenever it looks nice, or when we have (when we're transforming from unrelated types, so that there's not really anything that can be lerped individually). If we need more control about intermediate states, Lerp is the way to go.

Finally, notice that there is some inconvenience in the syntax once again. It definitely is really repetieive, but again this is something I'm very confident can be fixed with a language feature (maybe macros, or maybe something like cpp's std::exchange).
