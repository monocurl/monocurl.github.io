3. The Basics of Animation

The way animations are done is one of the more unique parts of monocurl. This section gives a motivation of why I chose the things I did. Strictly speaking, it's not fully necessary for coding in monocurl, but if you understand the decisions, they come across as natural rather than memorized.

3.1 Traditional Programatic Animation Methods

Traditional animation methods. The play method. But the thing is, this is pretty bad for composition. I'll not go into to much detail, but it actually becomes hard to express that function a should be in parallel with function b

This leads us more into the method monocurl time. What happens if we delay the time between animation creation and animation playing? Then, we can create the animations for a and b (not playing it yet), and then sometime later play it. This seems to solve all of our problems! The idea is that each slide, we create a list of animations to play, and then we play it all together. By separating the creation an execution, we are able to compose arbitrarily complex animations

I should mention this isn't the only solution, we can do something like coroutines. Of course, this was rather complex for me to write, and I think the perspective I have serves some other benefits as well.
