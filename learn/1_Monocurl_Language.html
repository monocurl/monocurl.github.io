1.1 The Basiscs

The basics: Monocurl comes as a pretty simple language. It is run line by line, so on each line we can give a command. In particular, we can declare a variable (which you can think of as a name for some data) by saying var x = 0 or let x = 0 to denote it is either mutable or immutable. There are a couple of datatypes. Numbers. AT this time no integers for simplicity. Vector, otherwise referred to as a list Strings, used very rarely but occasionally (for things like displaying text) Map, sometimes referred to as a dictionary. Meshes Animations Functions

Copy by value and that it is dynamically typed for the most part

We can do if statements, else if, and else statemenst, as well as for loops and while loops.

I'll gloss over some operators, but one that is extremely important is the vector plus operator

1.2 Functions

The rest of the article is devoted to functions and functors. Special importance is placed on these two since this is where Monocurl differs from many different languages.
Functions. Monocurl heavily emphasizes functions, and thus the syntax is very simple.

1.3 Functors

We'll discuss the full use case of functors later, but for motivation we can see that in many cases we just want to have the result of a function pretty much identical but change one value. Really, we need something that acts like the value returned, yet remembers how it was created so we can slightly alter that. We need to remember the function and its arguments.

Functors. This is really great because we do not have to memorize the argument order. It does make it a verbose, but verbose is good when you're learning!

Now, after creating the functor, we can reference its inputs and change them. Perhaps if you can see ahead, you can see how changing it slightly leads to an easy way to an animation. Maybe we can even interpolate functor arguments?

As of this writing, all functions can become functors. I later realized this is somewhat of a mistake, and this possibly may change in the future
